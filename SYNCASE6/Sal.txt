//
// This is the source for the case-sensitive syntax hiliting file
// for TSE Pro 4.4's macro language.
//
// It is intended to be used in combination with the SynCase macro,
// so that SAL words are automatically cased as they are edited or viewed,
// or so that existing SAL macros can be cased with Syncase.
//
// Important warning: never ever set TSE option IgnoreCaseOfKeyWords=TRUE,
// as this will lower-case the entire contents of the .syn file, and you
// will need (a backup of) this source file to repair the damage.
//
// How to install this file:
//
//    Open it in TSE.
//    Execute the macro SynCfg2.
//    You now have a Sal_cased.syn file without a directory.
//    SaveAs the file with a directory, optionally without "_cased".
//    If you retained the "_cased", then you might need to associate
//    the file extensions .s .si .ui .inc with the new sal_cased.syn file.
//    From now on SAL words will only be syntax hilited when correctly
//    cased, which the SynCase macro will automatically do for you.
//
// Other noteworthy differences with Semware's sal.syn file:
//
//    The # was removed as a directive character and added to the wordset.
//    Compiler directives were added as words. Benefit: now the latter are
//    only hilited when they exist and are correctly spelled.
//
//    Keyword group 6 consisted of a mishmash of keywords from other keyword
//    groups, so they were moved to those other keyword groups.
//
//    In SAL "break" is both a word (a statement) and a variable name, but we
//    cannot hilite them differently. Putting "break" in both keyword groups
//    doesn't work: only one hiliting color is used. So we have to make a
//    choice for a keyword group, and Semware and I made a different one.
//    Semware uses "break" mainly as a statement, and therefore put it in
//    keyword group 1. I am in the camp that never uses "goto", "break" out of
//    a loop, or "return" from the middle of a procedure, and therefore use
//    "break" only as a variable name and put it in keyword group 4.
//
// It is worth mentioning, that hiliting colors are assigned to keyword
// groups, not to languages. Therefore all languages (that you use) must
// assign the same meaning to the keyword groups. So if SAL is one of the
// languages that you use, then all your languages must assign the same
// meaning to the keyword groups as SAL.
//
// The SAL meaning of the keyword groups:
//
//    1  Words          predefined in and part of the language.
//    2  Characters     predefined in and part of the language.
//    3  Function names predefined in and part of the language.
//    4  Variable names predefined in and part of the language.
//    5  Constant names predefined in and part of the language.
//    6  Free
//    7  Free (*)
//    8  Free (*)
//    9  Free
//
//    (*) Semware uses keyword groups 7 and 8 for functions and variables
//        that are not released for general use. They are undocumented and
//        don't work in normal macros. They are just there as reserved words.
//        We are free to assign our own meaning to keyword groups 7 and 8.
//
